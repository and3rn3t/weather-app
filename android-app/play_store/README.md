# Play Store Metadata

This directory will contain all assets and metadata needed for Google Play Store submission.

## Required Assets

### App Icon

- **File:** `ic_launcher-playstore.png`
- **Size:** 512x512 px
- **Format:** PNG (32-bit)
- **Location:** Already in `app/src/main/res/` (generated by Android Studio)

### Feature Graphic

- **File:** `feature_graphic.png`
- **Size:** 1024x500 px
- **Format:** PNG or JPG
- **Purpose:** Displayed at the top of your Play Store listing
- **Requirements:**
  - No transparency
  - Showcases key features or branding

### Screenshots (Phone)

Minimum 2, maximum 8 screenshots required:

1. **01-main-weather-view.png** - Current weather display
2. **02-hourly-charts.png** - Hourly forecast with charts
3. **03-weather-radar.png** - Radar map view
4. **04-severe-alerts.png** - Severe weather alerts UI
5. **05-widgets.png** - Home screen widgets
6. **06-favorites.png** - Saved locations
7. **07-air-quality.png** - Air quality index
8. **08-settings.png** - Settings and themes

**Requirements:**

- Minimum dimension: 320px
- Maximum dimension: 3840px
- Aspect ratio: 16:9 or 9:16 (portrait recommended)
- Format: PNG or JPG
- 24-bit RGB (no alpha)

### Screenshots (Tablet) - Optional

- 7-inch and 10-inch tablet screenshots
- Same requirements as phone screenshots

### Promotional Video (Optional)

- YouTube video URL
- Showcases app features and UI

## Store Listing Text

All store listing text (titles, descriptions, etc.) is defined in `PlayStoreMetadata.json`.

### Short Description (80 characters max)

Appears in search results and browse pages.

### Full Description (4000 characters max)

Full app description with:

- Key features
- Benefits
- Use cases
- Privacy information

### Release Notes (500 characters max per language)

What's new in this version.

## Translations

To support multiple languages:

1. Create language-specific directories:

   ```
   play_store/
     listings/
       en-US/
       es-ES/
       fr-FR/
       de-DE/
       ja-JP/
   ```

2. Each directory should contain:
   - `title.txt` - App name (max 50 chars)
   - `short_description.txt` - Short description (max 80 chars)
   - `full_description.txt` - Full description (max 4000 chars)
   - `screenshots/` - Localized screenshots if needed

## Creating Screenshots

### Using Android Emulator

1. Launch emulator:

   ```bash
   # Create AVD if needed
   avdmanager create avd -n weather_test -k "system-images;android-34;google_apis;x86_64"
   
   # Launch emulator
   emulator -avd weather_test
   ```

2. Run the app:

   ```bash
   cd android-app
   ./gradlew installDebug
   ```

3. Take screenshots:
   - Use emulator's camera button
   - Or use ADB: `adb shell screencap -p /sdcard/screenshot.png`
   - Then pull: `adb pull /sdcard/screenshot.png`

### Using Fastlane Screengrab (Automated)

Install screengrab:

```bash
gem install fastlane
```

Set up test configurations and run:

```bash
fastlane screengrab
```

### Recommended Devices for Screenshots

- **Pixel 8 Pro** (1344x2992) - Flagship phone
- **Pixel 8** (1080x2400) - Standard phone
- **Pixel Tablet** (2560x1600) - Tablet (optional)

## Uploading to Play Console

### Option 1: Manual Upload

1. Go to [Google Play Console](https://play.google.com/console)
2. Select your app
3. Navigate to "Store presence" > "Main store listing"
4. Upload assets and fill in text fields

### Option 2: Using Fastlane

See `../../fastlane/README.md` for instructions on automated uploads.

### Option 3: Using Play Console API

The `PlayStoreMetadata.json` file can be used with Google Play Developer API for programmatic uploads.

## Content Rating

Complete the content rating questionnaire in Play Console:

- All answers are "No" except "Unrestricted Internet Access"
- Results in "Everyone" rating
- See `PlayStoreMetadata.json` for question answers

## Data Safety

Declare data collection practices:

- **Location data** - Used for app functionality (weather forecasts)
- **No data sharing** with third parties
- **Encrypted in transit** (HTTPS)
- **Optional** - users can manually search instead

See `PlayStoreMetadata.json` for complete data safety declaration.

## Testing Tracks

### Internal Testing

- Unlimited testers
- Immediate availability (no review)
- Great for QA team

### Closed Testing (Beta)

- Up to 100% of users
- Faster review than production
- Opt-in URL for testers

### Open Testing

- Public but marked as "early access"
- Anyone can join

### Production

- Full public release
- Subject to full review process

## Pre-Launch Report

Google Play automatically tests your app on:

- Real devices (various manufacturers)
- Different Android versions
- Multiple screen sizes

Review the pre-launch report before production release.

## Review Timeline

- **Internal track:** No review (instant)
- **Closed/Open track:** 1-3 days
- **Production track:** 1-7 days

## Checklist Before Submission

- [ ] App icon (512x512) uploaded
- [ ] Feature graphic (1024x500) created
- [ ] At least 2 phone screenshots
- [ ] Short description written (≤80 chars)
- [ ] Full description written (≤4000 chars)
- [ ] Content rating completed
- [ ] Data safety form filled
- [ ] Privacy policy URL provided
- [ ] APK/AAB uploaded and tested
- [ ] Target API level 34+ (Android 14)
- [ ] Signed with production keystore
- [ ] ProGuard enabled for release builds
